# Special Notes for Occlusion Experiment
# ---------------------------------------
# 1. Trigger remapping required for subjects 1-18
#    - Handled automatically via triggers_remapped.csv
#    - Located in: occlusion/docs/triggers_remapped.csv
#
# 2. Custom partitioner function for cross-condition generalization
#    - train_intact_test_occl.m must be on MATLAB path
#    - Tests invariance by training on intact, testing on occluded
#
# 3. Clean Rawdata plugin essential for infant data
#    - Infants have more motion artifacts
#    - Automatic bad channel detection critical
#
# 4. Memory considerations for RDM construction
#    - 4×4 identity RDM: ~6 pairwise comparisons
#    - Requires sufficient RAM for temporal generalization
#
# 5. GPU support optional but beneficial
#    - Speeds up permutation testing (100+ iterations)
#    - Not required but recommended for large-scale analyses

# Experimental Design Specifics
# ------------------------------
# Stimuli: 64 unique (4 identities × 2 colors × 8 variations)
# Occlusion: 50% intact, 50% occluded per block
# Occluders: Gabor patches (low/high spatial frequency, left/right position)
# Trial duration: 500ms stimulus + 800-1200ms ISI

# Data Quality Notes
# ------------------
# - Infant data: Expect 40-60% trial retention after artifact rejection
# - Adult data: Expect 80-95% trial retention
# - Minimum 1 valid block required for single-subject analysis
# - Recommended: 2+ blocks for reliable decoding estimates
